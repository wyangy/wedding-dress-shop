# Project Brief

## Project Title

Wedding Dress Shop

## Client Context

The client is a retired wedding dress designer with a remaining collection of bridal dresses from her design work.

The dresses were used as boutique designer pieces for runway shows, photoshoots, and client fittings. They are not ordinary privately owned second-hand dresses.

## Core Problem

The client wants to present and sell the remaining dress collection, but she needs a simple and clear online presence that makes the dresses easy to browse and understand.

Potential buyers need confidence before making contact because wedding dresses require accurate fit, clear condition details, and trustworthy presentation.

## Project Goal

The goal is to design and build a responsive bridal dress catalogue website.

The website should allow users to:

* Browse the dress collection
* View individual dress details
* Check measurements and condition notes
* Understand the story and value of the dresses
* Request a fitting or send an enquiry

## MVP Decision

The first version will not include cart, checkout, payment, or an automatic booking calendar.

Instead, the main user action is:

**Request a Fitting**

This is more realistic for wedding dresses because the buyer usually needs to ask questions, confirm availability, and discuss fit before making a decision.

## MVP Pages

* Homepage
* Shop / catalogue page
* Product detail page
* Request a Fitting page
* Contact / enquiry section

## Main User Flow

Home
→ Shop
→ Product Detail
→ Request a Fitting

## Important UX Note

Submitting the fitting request form does not confirm an appointment immediately.

It sends the buyer’s contact details and interest in a specific dress. The designer can then reply to confirm availability, fitting options, or next steps.

## Success Criteria

The project is successful if users can:

* Understand the purpose of the site quickly
* Browse available dresses easily
* Open a product detail page
* Find dress measurements and condition notes
* Request a fitting without confusion
* Use the site comfortably on desktop and mobile
